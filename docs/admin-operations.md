# 后台运营与 AI 使用说明

本文档记录当前后台的实际使用方式，重点覆盖 AI 改站、素材库 AI 描述、访问日志、备份恢复和上线安全项。

## 后台入口

- 后台：`/admin`
- 可视化编辑：`/admin/visual`
- 移动素材入口：`/admin/album`

首次打开后台时，如果没有配置环境变量账号，会进入管理员初始化流程。生产环境建议使用：

```bash
ADMIN_USERNAME=你的管理员名
ADMIN_PASSWORD=高强度密码
ADMIN_TOKEN_SECRET=至少32位随机字符串
YK_DATA_DIR=/opt/yakewangye-local
```

## AI 配置

后台路径：`AI 配置`

支持三种供应商：

- `Mock 本地占位`：不调用外部 AI，用于演示。
- `OpenAI Compatible`：兼容 Chat Completions 格式的供应商，例如部分 DeepSeek/OpenRouter/代理网关。
- `OpenAI Responses`：OpenAI Responses API 格式。

注意：

- API Key 默认在后台隐藏显示。
- 配置后先点“测试 AI 连接”。
- 文本改站可以使用文本模型。
- 截图改站、图片素材视觉理解需要支持图片输入的模型；如果模型不支持图片，素材分析会退回元数据分析。
- 视频素材当前不会逐帧理解，只基于文件名、标题、文件夹和上下文生成适用建议。

## AI 改内容

后台路径：`AI 改内容`

支持三种工作流：

- 只写自然语言描述，让 AI 生成草稿。
- `逐个素材 AI 改站`：按素材库顺序逐个调用 AI，每次只处理一个素材，累计生成最终草稿并显示进度。
- 上传 0 到多张截图，再写修改描述，让 AI 结合截图理解区域。
- 导出 AI 文案包给外部 AI，改完后导回差异预览。

安全边界：

- AI 不能直接覆盖网站。
- AI 结果必须进入差异预览，由管理员勾选后应用。
- 服务端只接受白名单 path。
- AI 可以通过受控动作新增服务和图册：
  - `create.services`：当素材库 AI 标注显示某张图片代表当前未覆盖的新牙科服务或服务场景时，新增服务项。
  - `create.gallery`：当素材更适合医院环境、住宿环境、路线/翻译支持、设备或视频展示时，新增图册项。
- 电话、地址、Telegram、API Key、模型配置、id、slug、href 等不允许被 AI 修改。
- 媒体字段只能使用素材库已有 URL，不能编造外链。

## 三语同步

后台顶部有 `中文一键翻译 RU/EN` 按钮。

- 按钮会读取当前后台草稿里的中文主内容，包括尚未保存的中文修改。
- 调用后台已配置的 AI，把中文内容翻译成俄语和英语覆盖层。
- 翻译完成后只写入当前后台草稿，仍需点击 `保存全部内容` 才会发布到网站。
- 翻译不会修改电话、地址、链接、素材 URL、AI 配置等固定信息。

## 素材库 AI 描述

后台路径：`素材库管理`

每个素材可以保存一份 AI 分析：

- 一句话摘要
- 视觉描述或元数据描述
- 标签
- 适合的使用场景
- 不适合的使用场景
- 建议放置位置
- 牙科/门诊相关性说明
- 可直接给访客看的说明
- 安全提示

按钮：

- `AI 分析`：只分析当前素材。
- `AI 分析未处理素材`：批量分析未分析或失败的素材，避免重复消耗 token。

AI 改站时会读取这些分析结果，选择更合适的图片/视频，写入服务图、医生图、文章封面、首页图册封面、图册视频素材等字段。若素材 AI 标注显示它代表现有网站没有覆盖的新服务或展示场景，AI 可以生成新增服务或新增图册草稿，仍需管理员在差异预览里确认。

素材较多时优先使用 `逐个素材 AI 改站`，避免一次请求过大或模型遗漏素材。这个流程会一个素材一次请求，失败的素材会跳过并记录错误，成功结果会继续累计到下一轮。

## AI 配置预设

后台路径：`AI 配置`

- 可以把当前供应商、API 地址、API Key、模型、Temperature、Max Tokens 保存为预设。
- 下次可从 `配置预设` 下拉框一键切换。
- 预设保存在内容数据里，仍会随后台备份一起导出。

## 访问日志

后台路径：`访问日志`

记录前台访客，不记录后台页面。当前采集：

- visitorId / sessionId
- 页面 URL 和路径
- 页面标题
- 来源 referrer / 搜索引擎
- 停留时间
- 语言、时区、屏幕与窗口尺寸
- IP
- User-Agent
- 浏览器、系统、设备类型

日志面板支持按 IP、浏览器、设备类型、手机/设备型号、系统、页面路径和关键词筛选。后台也会按 IP 合并访问，方便查看同一个 IP 的访问次数、会话数、页面访问数和最近访问时间。

隐私注意：

- 正式上线建议在隐私政策里说明会记录 IP、浏览器信息和访问路径。
- 当前日志保存在本地 JSON，适合早期运营查看；访问量大后应迁移到数据库或日志系统。

## 备份恢复

后台路径：`备份恢复`

后台备份包含：

- 内容 JSON
- 管理员配置
- 聊天记录
- 访问日志
- 素材库索引
- 上传文件

服务器整站备份仍由 `yk.sh` 负责。后台备份适合迁移/回滚内容，`yk.sh` 适合服务器级恢复。

## 上线安全清单

- 设置强随机 `ADMIN_TOKEN_SECRET`。
- 使用 HTTPS。
- 不要截图公开 API Key。
- 将 `YK_DATA_DIR` 放在代码目录外。
- 给 `/uploads` 做合理缓存，但不要允许目录列表。
- 控制 `VISUAL_AI_MAX_SCREENSHOT_MB` 和 `VISUAL_AI_MAX_SCREENSHOTS`，避免过大请求。
- 大量素材批量分析前先确认模型价格。
- 定期通过 `yk.sh` 创建全量备份，并验证可恢复。
